

# Fix Google OAuth — Code Update Plan

## Prerequisites (your side)
1. Add the two optional redirect URIs mentioned above to Google Cloud Console
2. Open Lovable Cloud → Users → Authentication Settings → Google
3. Select "Use your own credentials", paste your Client ID and Client Secret, save

## Code Changes (once credentials are configured)

### Step 1: Run Configure Social Auth tool
Generates `src/integrations/lovable/` module and installs `@lovable.dev/cloud-auth-js`.

### Step 2: Update `src/pages/Auth.tsx`
- `handleGoogleAuth`: Replace `signInWithOAuthCustomDomain({ provider: 'google', ... })` with `lovable.auth.signInWithOAuth("google", { redirect_uri: redirectUrl })`
- `handleAppleAuth`: Same swap for Apple
- `handleLinkedInAuth`: Keep as-is (LinkedIn not supported by Lovable Cloud)

### Step 3: Update `src/pages/InviteAcceptance.tsx`
- Same swap for Google and Apple providers in the OAuth handler

### Step 4: Update `src/pages/Settings.tsx` (Google Calendar)
- Keep the Supabase direct flow for calendar scope connections (`skipBrowserRedirect: true`) since it needs `calendar.readonly` scope
- This works because your own credentials are now configured with calendar scopes

### Step 5: Simplify `src/lib/oauth-helpers.ts`
- Narrow to LinkedIn-only helper (Google/Apple now go through Lovable Cloud module)

### Step 6: Remove `src/components/OAuthDiagnostics.tsx`
- No longer needed — was checking old direct config
