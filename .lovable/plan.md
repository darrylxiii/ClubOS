

# Phase 3: Long-term Code Optimizations

## Overview

Phase 3 focuses on structural code improvements that will provide **additional ~200-300MB build memory savings** and significantly faster cold starts for users.

---

## What We're Optimizing

### 3.1 Lazy-load @blocknote Editor (~200MB savings)

**Current State:**
- 16 files import from `@blocknote/*` directly
- The entire BlockNote library (~600KB) loads at module evaluation
- This includes `WorkspaceEditor.tsx` (381 lines) and 14 custom block components

**Solution:**
Create a `LazyBlockNoteEditor` wrapper that:
- Uses dynamic `import()` for `@blocknote/core`, `@blocknote/react`, `@blocknote/mantine`
- Shows a skeleton placeholder while loading
- Only loads when the workspace/editor route is accessed

**Files to Create/Modify:**
| File | Action |
|------|--------|
| `src/components/workspace/LazyWorkspaceEditor.tsx` | Create - wrapper with dynamic import |
| `src/components/workspace/WorkspaceEditor.tsx` | Modify - keep as implementation |
| Routes using WorkspaceEditor | Update to use lazy wrapper |

## Technical Details

### LazyWorkspaceEditor Pattern

```typescript
// Pattern for lazy-loaded editor
const WorkspaceEditorLazy = lazy(() => 
  import('./WorkspaceEditor').then(m => ({ default: m.WorkspaceEditor }))
);

export function LazyWorkspaceEditor(props: WorkspaceEditorProps) {
  return (
    <Suspense fallback={<EditorSkeleton />}>
      <WorkspaceEditorLazy {...props} />
    </Suspense>
  );
}
```

### DynamicChart Migration Pattern

```typescript
// Before (direct import)
import { BarChart, Bar, XAxis, YAxis } from 'recharts';
<BarChart data={data}>
  <XAxis dataKey="name" />
  <Bar dataKey="value" fill="#8884d8" />
</BarChart>

// After (DynamicChart)
import { DynamicChart } from '@/components/charts/DynamicChart';
<DynamicChart 
  type="bar" 
  data={data}
  config={{
    xAxisKey: 'name',
    bars: [{ dataKey: 'value', fill: '#8884d8' }]
  }}
/>
```

## Benefits Beyond Memory

1. **Faster Initial Page Load**: Charts and editor only load when needed
2. **Better Code Splitting**: Vite can create optimal chunks
3. **Improved User Experience**: Skeleton loaders while heavy components load
4. **Future-Proof**: Easier to upgrade recharts or blocknote versions
5. **Consistent Patterns**: All charts use same DynamicChart API
